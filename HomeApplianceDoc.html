Windows form application
with Relational database

This is a personal project of mine to develop a simple offline windows form application.
In this project, C# and microsoft sql are mainly used.
A showcase of database queries and OOP skills.

Project Overview

It is a simple database windows application.

Login form
The first form to appear when the application is running.

There are two user types which can be choosen to login
Different user types have different authority on using the application.

By the way,the Admin account is 'Admin' and password is 'Admin123'.

There are validation and exception implemented in this form in which you can check in the git repository.
If the user has failed to login for a certain attempts, the user have to wait a certain amount of time.

Whenever a new form is instantiated from login form, the login form class will be hidden. When the instantiated classes are closed, original login form object must be shown again. Instead of instantiating new login classes, it is more practical to reuse initial login instance.

Registration process

To use the application in customer position, an account is required.
There are also restrictions and validation processes.

The user name must only include letters and numbers. No special characters are allowed.
Password must be between 8 to 16 characters.
Passowrd must include at least one uppercase or lowercase character.
Of course, password and confirm password must match.

Admin Form

Admin has the authority to create, read, modify and delete appliance data from the database.
Double click on a record and respected data will be filled in each text box. And then, the admin can modify the information.

There are validations for checking if the admin input the wrong data types in certain fields.

Customer Form

Basically, the customer can view the applicances.

If the users want to rend some appliance(s), they can select the applicance and choose the rental period to rend and click add to cart.
The choosen rental period must be greater than minimum rental period.
The item count in the cart and the total expense will be shown.
